## GET방식과 POST방식

- HTTP 프로토콜을 이용해서 서버에 무언가를 전달할 때 사용하는 방식이다.

## GET 방식

> 주로 **데이터를 조회**할 경우 사용(SELECT)

- Get 방식은 어떠한 정보를 가져와 조회할 때 사용하는 방식이다.
- Get 방식은 **URL에 파라미터를 포함시켜 요청**을 한다. 전송하는 길이에 제한이 있다.
- **캐싱이 가능**하다. Post 방식보다 속도면에서 빠르다.

## POST 방식

> 주로 **데이터를 추가, 수정**할 경우 사용(INSERT, UPDATE, DELETE)

- Post 방식은 데이터를 서버로 보내 추가, 수정하기 위해 사용하는 방식이다.
- Get 방식과 다르게 **URL에 데이터가 노출되지 않는다.**
- 데이터를 Body에 포함한다.
- 헤더필드 중 Body 데이터를 설명하는 Content-Type이라는 헤더 필드가 들어가고 어떤 데이터 타입인지 명시해야한다.

### 대표적인 콘텐츠 타입

1. application/
2. text/
3. multipart/

## 차이점

- Get 방식은 조회할 때 사용하고 Post방식은 추가, 수정할 때 사용한다.
- Get 방식은 URL에 데이터를 붙여 보내고, Post 방식은 Body에 데이터를 넣어서 보낸다.
  - 💡 보안적으로 Get방식이든 Post방식이든 추가로 조치가 필요하다.
